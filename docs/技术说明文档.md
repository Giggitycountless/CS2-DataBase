## 1. 项目整体架构

应用程序分为四个层次，严格自上而下通信：

```
┌─────────────────────────────────────────────────────────┐
│                        UI 层                            │
│  MainFrame  CrudPanel  AnalyticsPanel  Styles           │
│  （Swing / EDT 事件派发线程）                            │
└───────────────────────┬─────────────────────────────────┘
                        │  调用
                        ▼
┌─────────────────────────────────────────────────────────┐
│                  数据契约（接口）                        │
│                   AppRepository                         │
└──────────┬─────────────────────────────┬────────────────┘
           │ 实现                        │ 实现
           ▼                             ▼
┌──────────────────────┐   ┌──────────────────────────────┐
│ CounterStrike        │   │ MongoCounterStrike            │
│ Repository           │   │ Repository                   │
│（Oracle / JDBC）     │   │（MongoDB Driver）            │
└──────────┬───────────┘   └──────────────┬───────────────┘
           │ 使用                          │ 使用
           ▼                              ▼
┌──────────────────────┐   ┌──────────────────────────────┐
│   Database.java      │   │   MongoDatabase.java         │
│  （JDBC 连接管理）   │   │  （MongoClient 连接池）      │
└──────────────────────┘   └──────────────────────────────┘
```

**核心设计原则 — 接口隔离：**
UI 层持有的是 `AppRepository`（接口）的引用，从不直接引用具体实现类。这意味着 UI 层不知道、也不关心数据来自 Oracle 还是 MongoDB。运行时切换后端，只需把一个不同的实现对象传给 `MainFrame` 即可，界面代码完全不用改。

## 2. 程序入口 — CounterStrikeApp

**文件：** `src/main/java/com/counterstrike/app/CounterStrikeApp.java`

### `main()` — JVM 入口

```java
public static void main(String[] args) {
    FlatDarkLaf.setup();                      // 安装深色主题
    UIManager.put("Button.arc", 8);           // 按钮圆角
    UIManager.put("ScrollBar.thumbArc", 999); // 圆形滚动条滑块
    // ... 更多 FlatLaf 细节设置 ...
    SwingUtilities.invokeLater(() -> connect(null));
}
```

`SwingUtilities.invokeLater()` 在 Java Swing 中是强制要求。Swing **不是线程安全的**：所有 UI 的构建和操作必须在 **EDT（事件派发线程）** 上执行。`invokeLater` 将 Lambda 调度到 EDT 上，在 JVM 完成初始化后执行。传入 `null` 表示这是初次启动（没有需要关闭的旧窗口）。

### `connect(JFrame oldFrame)` — 连接对话框与对象组装

```java
public static void connect(javax.swing.JFrame oldFrame) {
    // 第一步：弹出对话框让用户选择后端
    String[] options = {"Oracle (Relational)", "MongoDB (NoSQL)"};
    int choice = JOptionPane.showOptionDialog(oldFrame, ...);

    if (choice < 0) return;  // 用户关闭了对话框——不做任何事

    try {
        AppRepository repository;
        String dbInfo;

        if (choice == 0) {
            // --- Oracle 路径 ---
            DatabaseConfig config = DatabaseConfig.load(); // 读取 app.properties
            Database database = new Database(config);
            database.testConnection();                     // 连不上立即报错
            repository = new CounterStrikeRepository(database);
            dbInfo = config.url();

        } else {
            // --- MongoDB 路径 ---
            String uri = (String) JOptionPane.showInputDialog(...); // 提示输入 URI
            if (uri == null || uri.isBlank()) return;
            MongoDatabase mongo = new MongoDatabase(uri.trim(), "cs_small");
            mongo.testConnection();                        // 发送 ping 命令验证
            repository = new MongoCounterStrikeRepository(mongo);
            dbInfo = uri.trim() + "/cs2";
        }

        // 第二步：构建 UI，传入选好的 repository
        MainFrame frame = new MainFrame(repository, dbInfo);
        frame.setVisible(true);

        // 第三步：若是切换数据库，关闭旧窗口
        if (oldFrame != null) oldFrame.dispose();

    } catch (Exception exception) {
        // 显示错误。连接失败时 oldFrame 仍然存活、可正常使用。
        JOptionPane.showMessageDialog(oldFrame, exception.getMessage(), ...);
    }
}
```

**为什么是 `public static`？**
`MainFrame` 里的"Switch DB"按钮调用 `CounterStrikeApp.connect(MainFrame.this)`。由于 `MainFrame` 在不同的包（`ui`），该方法必须是 `public` 才能被访问。设为 `static` 则不需要 `CounterStrikeApp` 实例——它本质上是一个纯工厂/工具方法。

**快速失败设计：** `testConnection()` 在创建 `MainFrame` 之前调用。如果数据库不可达，异常立刻被捕获并弹出错误对话框，不会显示一个半成品窗口。

## 3. 配置层 — DatabaseConfig

**文件：** `src/main/java/com/counterstrike/app/config/DatabaseConfig.java`

```java
public record DatabaseConfig(String url, String user, String password) {
    public static DatabaseConfig load() { ... }
}
```

`DatabaseConfig` 是 Java **record**——一种不可变的数据容器。三个字段（`url`、`user`、`password`）对应 `app.properties` 中的键：

```
db.url      = jdbc:oracle:thin:@localhost:1521/FREE
db.user     = system
db.password = oracle
```

### 配置文件查找顺序（`findConfigPath()` 方法）

加载器按顺序尝试以下四个位置，找到第一个存在的就停止：

1. **系统属性** `-Dcs2.config=C:\path\to\app.properties`（命令行传入）
2. **环境变量** `CS2_DB_CONFIG`
3. **工作目录** `./app.properties`（JAR 文件旁边）
4. **Classpath** `/app.properties`（打包在 JAR 内部，最终兜底）

这种分层查找使同一个 JAR 文件能在开发（磁盘文件）、CI（环境变量）、独立发布（内嵌配置）三种场景下无缝工作，无需修改任何代码。

## 4. 数据库连接层

### 4.1 Database.java（Oracle / JDBC）

**文件：** `src/main/java/com/counterstrike/app/db/Database.java`

```java
public final class Database {
    private final DatabaseConfig config;

    public Connection getConnection() throws SQLException {
        loadOracleDriverIfPresent();
        return DriverManager.getConnection(config.url(), config.user(), config.password());
    }

    public void testConnection() throws SQLException {
        try (Connection ignored = getConnection()) {
            // 打开并立即关闭——验证 URL、凭证和 JDBC 驱动是否正常
        }
    }

    private void loadOracleDriverIfPresent() {
        try {
            Class.forName("oracle.jdbc.OracleDriver");
        } catch (ClassNotFoundException ignored) {
            // JDBC 4+ 驱动通过 ServiceLoader 自动注册，这里只是备用方案
        }
    }
}
```

**重要设计选择——不使用连接池：**
每次调用 `getConnection()` 都会打开一个**全新的 JDBC 连接**。每个 repository 方法调用 `getConnection()`，使用连接，然后在 `try-with-resources` 块里关闭它。对于单用户桌面应用来说，这简单且正确。生产级服务器会使用连接池（HikariCP、c3p0 等）来复用连接，但在这里开销可以接受，简洁性更有价值。

**`loadOracleDriverIfPresent()` 的作用：** JDBC 4.0（Java 6+）引入了通过 `java.util.ServiceLoader` 自动发现驱动的机制。驱动 JAR 包含 `META-INF/services/java.sql.Driver` 文件，可自动注册。显式调用 `Class.forName()` 是针对旧环境的双重保险。

### 4.2 MongoDatabase.java（MongoDB）

**文件：** `src/main/java/com/counterstrike/app/db/MongoDatabase.java`

```java
public final class MongoDatabase {
    private final String uri;
    private final String dbName;
    private MongoClient client;  // 首次使用前为 null

    public com.mongodb.client.MongoDatabase getDatabase() {
        if (client == null) {
            client = MongoClients.create(uri);  // 懒加载：第一次调用才建连接池
        }
        return client.getDatabase(dbName);
    }

    public void testConnection() {
        getDatabase().runCommand(new Document("ping", 1));
    }

    public void close() {
        if (client != null) {
            client.close();  // 关闭连接池及所有后台线程
            client = null;
        }
    }
}
```

**懒加载：** `MongoClients.create(uri)` 创建连接池（多个 socket、心跳线程、服务器监控线程），开销较大，所以推迟到第一次实际数据库调用时才执行，而不是在构造时。

**`getDatabase(dbName)` 不发起网络请求。** 它只是返回一个 `MongoDatabase` 句柄对象。真正的网络 I/O 在访问集合并执行命令时才发生。

**`testConnection()` 发送 `ping`：** MongoDB 的 `ping` 命令是最小化的服务器往返测试，如果服务器可达且凭证有效，返回 `{"ok": 1}`。

**`close()` 对切换数据库功能至关重要：** 用户切换数据库时，旧的 `MongoDatabase` 必须关闭，以释放后台线程池和 socket 连接。如果从不调用 `close()`，每次切换都会泄漏一个 `MongoClient` 及其关联的系统资源，直到 JVM 退出。

## 6. 元数据层 — Schema、TableSpec、ColumnSpec

这一层是整个"一次编写、到处复用"CRUD 系统的核心。数据库中的每张表都被描述为一个 Java 对象。Repository 用这些描述自动生成 SQL，UI 用它们自动生成表单，无需为每张表单独写代码。

### 6.1 ColumnSpec — 描述单个列

**文件：** `src/main/java/com/counterstrike/app/repository/ColumnSpec.java`

```java
public record ColumnSpec(
    String name,          // 数据库字段名，如 "PersonID"
    String label,         // 界面显示名，如 "Person ID"
    Type type,            // TEXT | INTEGER | DECIMAL | DATE
    boolean primaryKey,   // true → 该列是主键（编辑时不可修改）
    boolean required,     // true → 界面标记 * 并验证非空
    ForeignKey foreignKey // 非 null → 界面渲染可编辑下拉框
) {
    public enum Type { TEXT, INTEGER, DECIMAL, DATE }
    public record ForeignKey(String table, String column) {}
    public boolean isDate() { return type == Type.DATE; }
}
```

**工厂方法** 简化了冗长的构造器调用：

| 方法 | 用途 |
|---|---|
| `pk(name, label, type)` | 主键列（primaryKey=true，required=true，无外键） |
| `pkRef(name, label, type, table, col)` | 既是主键又是外键（如 Player.PersonID → People.PersonID） |
| `col(name, label, type)` | 可选的普通非主键列 |
| `required(name, label, type)` | 必填的非主键列 |
| `ref(name, label, type, table, col, required)` | 有外键引用的非主键列 |

**`ForeignKey` 如何驱动下拉框：**
`CrudPanel` 打开添加/编辑表单时，对每个 `foreignKey` 非空的列调用 `repository.distinctValues(fk.table(), fk.column())`。结果被加载到**可编辑的** `JComboBox` 中，用户可以从已有值中选择，也可以自由输入新值。

**国籍的自引用：**
```java
ref("Nationality", "Nationality", Type.TEXT, "People", "Nationality", false)
```
这告诉 `CrudPanel` 调用 `distinctValues("People", "Nationality")`，执行 `SELECT DISTINCT Nationality FROM People`。结果就是数据库中已有的国籍列表，作为建议选项显示。由于调用了 `JComboBox.setEditable(true)`，用户仍然可以输入任何新国籍——没有强制约束。

### 6.2 TableSpec — 描述一张表

**文件：** `src/main/java/com/counterstrike/app/repository/TableSpec.java`

```java
public record TableSpec(
    String displayName,      // 界面显示名，如 "Player Contract"
    String tableName,        // 真实 SQL 表名，如 "PlayerContract"
    List<ColumnSpec> columns,
    String orderBy           // 默认 ORDER BY 子句
) {
    public List<ColumnSpec> primaryKey()        // → primaryKey == true 的列
    public List<ColumnSpec> insertableColumns() // → 所有列（INSERT 使用）
    public List<ColumnSpec> editableColumns()   // → 非主键列（UPDATE SET 使用）
    public String[] columnLabels()              // → label 字符串数组（JTable 表头）
}
```

`insertableColumns()` 返回**所有**列，因为插入新行时需要为每个字段提供值，包括主键。`editableColumns()` 只返回非主键列，因为不能修改主键——主键在 WHERE 子句中用于定位该行。

**`orderBy`** 是拼接到每个 `browse()` SELECT 末尾的原始 SQL 片段：
```java
// Tournament 表——最新的排在前面，ID 保证稳定排序
new TableSpec("Tournament", "Tournament", ...,
              "TournyStartDate DESC NULLS LAST, TournamentID DESC")
```

`NULLS LAST` 是 Oracle 语法，将 NULL 日期的行排在最后而不是最前。

### 6.3 Schema — 全部表的目录

**文件：** `src/main/java/com/counterstrike/app/repository/Schema.java`

```java
public static final List<TableSpec> ALL = List.of(
    PEOPLE, PLAYER, COACH, TEAM,
    PLAYER_CONTRACT, COACH_CONTRACT,
    TOURNAMENT, MATCH, MATCH_RECORD,
    PLAYER_MATCH_STATS, TOURNAMENT_PARTICIPATION
);
```

`Schema.ALL` 被 `CrudPanel` 遍历，用于填充表格选择器下拉框。因为 `TableSpec` 通过 `displayName` 作为 `toString()` 返回值，`JComboBox` 会自动显示友好名称。

**这个设计为什么强大：**
向应用程序添加一张新表，只需要**修改一处**：在 `Schema.java` 里加一个 `TableSpec` 常量。Repository 的 `browse()`、`insertRow()`、`updateRow()`、`deleteRow()` 方法会自动处理它，`CrudPanel` 会自动将它添加到表格选择器并渲染正确的表单。

## 8. Oracle 实现 — CounterStrikeRepository

**文件：** `src/main/java/com/counterstrike/app/repository/CounterStrikeRepository.java`

### 8.1 搜索如何工作

每个 `find*()` 方法都遵循相同模式。以 `findTopPlayers()` 为例逐步分析：

```java
public TableData findTopPlayers(String search) throws SQLException {
    String sql = """
        SELECT p.PersonID, p.Nickname, p.FullName, pl.Rating, pl.ADR, pl.DPR
        FROM People p
        JOIN Player pl ON p.PersonID = pl.PersonID
        WHERE LOWER(p.FullName) LIKE ?
           OR LOWER(NVL(p.Nickname, '')) LIKE ?
           OR TO_CHAR(pl.Rating) LIKE ?
           OR TO_CHAR(pl.ADR)    LIKE ?
           OR TO_CHAR(pl.DPR)    LIKE ?
        ORDER BY pl.Rating DESC NULLS LAST, p.FullName
        """;
    String textLike = like(search);   // "s1mple" → "%s1mple%"
    return queryTable(
        new String[]{"PlayerID","Nickname","Name","Rating","ADR","DPR"},
        sql,
        statement -> {
            statement.setString(1, textLike);
            statement.setString(2, textLike);
            statement.setString(3, numberLike(search)); // 匹配 "1.0" 或 "1"
            statement.setString(4, numberLike(search));
            statement.setString(5, numberLike(search));
        }
    );
}
```

**`like(String search)` 将用户输入转换：**
- 空字符串 → `"%"`（匹配所有——`WHERE col LIKE '%'` 永远为真）
- 否则 → `"%" + search.trim().toLowerCase() + "%"`

**`numberLike(String search)` 处理数字格式：** 同上，但还会去掉末尾的 `.0`，确保搜索 "1" 能找到存储为 "1.0" 的值。

**`LOWER(p.FullName) LIKE ?`：** 同时对列值和搜索词都转小写，实现大小写不敏感匹配，无需 Oracle 的大小写不敏感排序规则。

**`NVL(p.Nickname, '')`：** `NVL` 是 Oracle 的空值合并函数。如果不用它，`LOWER(NULL) LIKE '%s1mple%'` 的结果是 `NULL`（而不是 `FALSE`），意味着昵称为 NULL 的行永远不会匹配——即便用户没有搜索任何具体内容。用 `NVL(..., '')` 把 NULL 转为空字符串，`LIKE` 比较才能正常工作。

**`TO_CHAR(pl.Rating)`：** Rating 是 `NUMBER` 列。Oracle 不能对数字直接使用 `LIKE`，所以先转为字符串。这使搜索栏能通过输入 "1" 或 "05" 来匹配 "1.05"。

### 8.2 浏览如何工作（元数据驱动的 SELECT）

`browse()` 是 Manage 页面的引擎。它从 `TableSpec` 自动生成完整的 SELECT 语句，无需任何针对特定表的代码：

```java
public TableData browse(TableSpec spec, String search) throws SQLException {
    List<ColumnSpec> columns = spec.columns();

    // ── 从列名构建 SELECT 子句 ───────────────────────────────────────────
    StringJoiner select = new StringJoiner(", ");
    for (ColumnSpec column : columns) {
        select.add(column.name());
    }
    // 结果："PersonID, Nickname, FullName, Birthday, Nationality"

    // ── 构建 WHERE 子句——每列一个 LIKE 条件 ────────────────────────────
    StringBuilder sql = new StringBuilder("SELECT ").append(select)
            .append(" FROM ").append(spec.tableName());

    String trimmed = normalize(search);
    boolean filtered = !trimmed.isEmpty();
    if (filtered) {
        StringJoiner where = new StringJoiner(" OR ", " WHERE ", "");
        for (ColumnSpec column : columns) {
            if (column.isDate()) {
                // 日期必须先格式化为文本才能用 LIKE 匹配
                where.add("TO_CHAR(" + column.name() + ", 'YYYY-MM-DD') LIKE ?");
            } else {
                where.add("LOWER(TO_CHAR(" + column.name() + ")) LIKE ?");
            }
        }
        sql.append(where);
    }

    // ── 附加来自 TableSpec 元数据的 ORDER BY ────────────────────────────
    sql.append(" ORDER BY ").append(spec.orderBy());

    // ── 为每列绑定一个参数 ───────────────────────────────────────────────
    String textLike = like(trimmed);
    String dateLike = numberLike(trimmed);
    return queryTable(spec.columnLabels(), sql.toString(), statement -> {
        if (filtered) {
            int index = 1;
            for (ColumnSpec column : columns) {
                statement.setString(index++, column.isDate() ? dateLike : textLike);
            }
        }
    });
}
```

**示例：以 "russia" 搜索 People 表**

生成的 SQL：
```sql
SELECT PersonID, Nickname, FullName, Birthday, Nationality
FROM People
WHERE LOWER(TO_CHAR(PersonID)) LIKE '%russia%'
   OR LOWER(TO_CHAR(Nickname)) LIKE '%russia%'
   OR LOWER(TO_CHAR(FullName)) LIKE '%russia%'
   OR TO_CHAR(Birthday, 'YYYY-MM-DD') LIKE '%russia%'
   OR LOWER(TO_CHAR(Nationality)) LIKE '%russia%'
ORDER BY PersonID
```

只返回任意列包含 "russia"（不区分大小写）的行。由于 Nationality 列包含 "Russia"，这些行会匹配。

**为什么到处用 `TO_CHAR(column)`？** Oracle 的 `LOWER()` 和 `LIKE` 只对字符串类型有效。`TO_CHAR()` 将日期和数字转为字符串，使同一个通用 WHERE 模板对所有列有效，无论其数据类型。

### 8.3 INSERT 如何工作

```java
public void insertRow(TableSpec spec, Map<String, String> rawValues) throws SQLException {
    List<ColumnSpec> columns = spec.insertableColumns();  // 所有列

    // 构建列名列表和占位符列表
    StringJoiner names = new StringJoiner(", ");
    StringJoiner marks = new StringJoiner(", ");
    for (ColumnSpec column : columns) {
        names.add(column.name());
        marks.add("?");
    }

    // 示例结果：INSERT INTO People (PersonID, Nickname, FullName, Birthday, Nationality)
    //            VALUES (?, ?, ?, ?, ?)
    String sql = "INSERT INTO " + spec.tableName()
               + " (" + names + ") VALUES (" + marks + ")";

    executeWrite(sql, statement -> {
        int index = 1;
        for (ColumnSpec column : columns) {
            bind(statement, index++, column, rawValues.get(column.name()));
        }
    });
}
```

**`bind(statement, index, column, rawValue)` 根据列类型将字符串转为正确的 Java/JDBC 类型：**

```java
private void bind(PreparedStatement stmt, int i, ColumnSpec col, String raw)
        throws SQLException {
    if (raw == null || raw.isBlank()) {
        stmt.setNull(i, Types.NULL);
        return;
    }
    switch (col.type()) {
        case INTEGER -> stmt.setLong(i, Long.parseLong(raw.trim()));
        case DECIMAL -> stmt.setBigDecimal(i, new BigDecimal(raw.trim()));
        case DATE    -> {
            LocalDate d = LocalDate.parse(raw.trim()); // 解析 "2003-05-15"
            stmt.setDate(i, java.sql.Date.valueOf(d));
        }
        case TEXT    -> stmt.setString(i, raw.trim());
    }
}
```

这种类型安全的绑定确保用户输入的 "1.05" 在数据库中成为 `BigDecimal(1.05)` 而不是字符串 "1.05"，"2003-05-15" 成为 `DATE` 类型值而不是字符串。

### 8.4 UPDATE 如何工作

```java
public void updateRow(TableSpec spec,
                      Map<String, String> primaryKey,
                      Map<String, String> rawValues) throws SQLException {

    List<ColumnSpec> editable = spec.editableColumns();  // 仅非主键列
    List<ColumnSpec> keys     = spec.primaryKey();

    // 构建：UPDATE People SET Nickname = ?, FullName = ?, Birthday = ?, Nationality = ?
    StringJoiner set = new StringJoiner(", ");
    for (ColumnSpec col : editable) set.add(col.name() + " = ?");

    // 构建：WHERE PersonID = ?
    StringJoiner where = new StringJoiner(" AND ");
    for (ColumnSpec key : keys) where.add(key.name() + " = ?");

    String sql = "UPDATE " + spec.tableName() + " SET " + set + " WHERE " + where;

    int affected = executeWrite(sql, statement -> {
        int index = 1;
        // 先绑定 SET 值
        for (ColumnSpec col : editable) {
            bind(statement, index++, col, rawValues.get(col.name()));
        }
        // 再绑定 WHERE 值（从选中行读出的主键原值）
        for (ColumnSpec key : keys) {
            bind(statement, index++, key, primaryKey.get(key.name()));
        }
    });

    if (affected == 0) {
        throw new IllegalArgumentException("No matching row was found to update.");
    }
}
```

`primaryKey` 是 `CrudPanel` 在打开编辑表单之前从**当前选中的表格行**读取的 `Map<String, String>`，包含原始主键值，用于 WHERE 子句精确定位要更新的那一行。

`affected == 0` 说明该行在用户选中到提交表单之间消失了（例如被其他会话删除）。这个错误会通知给用户。

### 8.5 DELETE 如何工作

```java
public void deleteRow(TableSpec spec, Map<String, String> primaryKey) throws SQLException {
    List<ColumnSpec> keys = spec.primaryKey();
    StringJoiner where = new StringJoiner(" AND ");
    for (ColumnSpec key : keys) {
        where.add(key.name() + " = ?");
    }
    // 示例：DELETE FROM People WHERE PersonID = ?
    String sql = "DELETE FROM " + spec.tableName() + " WHERE " + where;

    int affected = executeWrite(sql, statement -> {
        int index = 1;
        for (ColumnSpec key : keys) {
            bind(statement, index++, key, primaryKey.get(key.name()));
        }
    });

    if (affected == 0) {
        throw new IllegalArgumentException("No matching row was found to delete.");
    }
}
```

WHERE 子句由所有主键列用 `AND` 连接。对于复合主键（如 `PlayerMatchStats` 有 `MatchRecordID + PlayerID`），两个条件必须同时满足。

### 8.6 distinctValues — 下拉框数据来源

```java
public List<String> distinctValues(String table, String column) throws SQLException {
    String sql = "SELECT DISTINCT " + column
               + " FROM " + table
               + " WHERE " + column + " IS NOT NULL"
               + " ORDER BY " + column;
    List<String> values = new ArrayList<>();
    try (Connection connection = database.getConnection();
         PreparedStatement statement = connection.prepareStatement(sql);
         ResultSet resultSet = statement.executeQuery()) {
        while (resultSet.next()) {
            values.add(String.valueOf(resultSet.getObject(1)));
        }
    }
    return values;
}
```

`SELECT DISTINCT` 每个唯一值只返回一次。Oracle 在服务器端处理去重，只传输不重复的值——而不是所有行。`IS NOT NULL` 过滤器防止 null 值出现在下拉框中。

**示例——国籍下拉框：**
```sql
SELECT DISTINCT Nationality FROM People
WHERE Nationality IS NOT NULL
ORDER BY Nationality
-- 返回：["Australia", "Denmark", "France", "Russia", ...]
```

这些成为 `JComboBox` 的选项。由于调用了 `setEditable(true)`，用户可以输入列表中没有的新国籍。

### 8.7 executeWrite 的事务安全性

```java
private int executeWrite(String sql, StatementBinder binder) throws SQLException {
    try (Connection connection = database.getConnection()) {
        connection.setAutoCommit(false);   // 开启显式事务
        try (PreparedStatement statement = connection.prepareStatement(sql)) {
            binder.bind(statement);
            int affected = statement.executeUpdate();
            connection.commit();           // 持久化变更
            return affected;
        } catch (SQLException exception) {
            safeRollback(connection);      // 任何 SQL 错误都回滚
            throw new SQLException(friendlyMessage(exception), exception);
        } catch (RuntimeException exception) {
            safeRollback(connection);      // 编程错误也回滚
            throw exception;
        }
    }
}
```

JDBC 连接默认 `autoCommit = true`，即每条语句立即提交。设置 `autoCommit(false)` 开启**显式事务**：

- `executeUpdate()` 成功 → `commit()` 使变更永久生效
- 发生任何异常 → `rollback()` 撤销本次连接中的所有变更

`friendlyMessage(exception)` 将 Oracle 错误码翻译为可读文本：
```java
private String friendlyMessage(SQLException e) {
    return switch (e.getErrorCode()) {
        case 1    -> "该 ID 的记录已存在（唯一约束冲突）。";
        case 2291 -> "引用的记录不存在（外键约束违反）。";
        case 2292 -> "无法删除——其他记录依赖于此行。";
        case 1400 -> "必填字段缺失（NOT NULL 约束违反）。";
        default   -> e.getMessage();
    };
}
```

Oracle 错误 2291（ORA-02291）表示"父键不存在"——用户尝试插入一个 PlayerContract，引用了 People 中不存在的 PersonID。

### 8.8 SQL 注入防护

所有用户提供的值都通过 JDBC **PreparedStatement** 的 `?` 占位符绑定，从不进行字符串拼接。这是防止 SQL 注入的根本措施。

```java
// 安全 —— 用户输入作为值绑定，永远不会被解释为 SQL
statement.setString(1, userInput);

// 危险 —— 本代码库中从不这样做
String sql = "SELECT * FROM People WHERE Name = '" + userInput + "'";
```

如果 `userInput` 是 `' OR '1'='1`，安全版本将其作为字面字符串传给 Oracle；危险版本会将 WHERE 子句变成 `WHERE Name = '' OR '1'='1'`，返回所有行。

`browse()`、`insertRow()`、`updateRow()`、`deleteRow()` 中的列名和表名来自 `Schema.java`（编译时常量），而非用户输入，因此也不存在注入风险。

### 8.9 分析查询

```java
public TableData runAnalytics(AnalyticsQuery query, String parameter) throws SQLException {
    String value = like(parameter);                         // "%param%" 或 "%"
    int placeholders = query.placeholderCount();            // 统计 SQL 中 "?" 的数量
    try (Connection connection = database.getConnection();
         PreparedStatement statement = connection.prepareStatement(query.sql())) {
        for (int index = 1; index <= placeholders; index++) {
            statement.setString(index, value);              // 所有 "?" 绑定同一个值
        }
        try (ResultSet resultSet = statement.executeQuery()) {
            return readResultSet(resultSet);
        }
    }
}
```

`readResultSet()` 通过 `ResultSetMetaData` 动态读取列名，分析查询结果无需预先声明列列表：

```java
private TableData readResultSet(ResultSet resultSet) throws SQLException {
    ResultSetMetaData meta = resultSet.getMetaData();
    int cols = meta.getColumnCount();
    String[] columns = new String[cols];
    for (int i = 0; i < cols; i++) {
        // getColumnLabel 返回 AS 别名；下划线替换为空格
        columns[i] = meta.getColumnLabel(i + 1).replace('_', ' ');
    }
    List<Object[]> rows = new ArrayList<>();
    while (resultSet.next()) {
        Object[] row = new Object[cols];
        for (int i = 0; i < cols; i++) {
            row[i] = resultSet.getObject(i + 1);
        }
        rows.add(row);
    }
    return new TableData(columns, rows);
}
```

SQL 中的列别名如 `Avg_Rating` 在表头显示时变为 "Avg Rating"（下划线替换为空格）。

## 9. MongoDB 实现 — MongoCounterStrikeRepository

### 9.1 文档结构设计

`cs_small` MongoDB 数据库采用**嵌套文档**设计——相关数据嵌套在父文档内，而不是存储在独立集合中：

```
集合：people
{
  _id: 1,                          ← PersonID（Long 类型）
  nickname: "s1mple",
  fullName: "Oleksandr Kostyliev",
  birthday: ISODate("2003-05-05"),
  nationality: "Ukraine",
  type: "player",                   ← "player"、"coach" 或 "other"
  playerStats: {                    ← 嵌套的 Player 表数据
    rating: 1.35,
    adr: 82.1,
    dpr: 0.58
  },
  contract: {                       ← 嵌套的 PlayerContract 数据
    teamName: "Natus Vincere",
    startDate: ISODate("2022-01-01"),
    endDate: ISODate("2024-12-31"),
    inGameRole: "Rifler"
  }
}

集合：matches
{
  _id: 101,
  tournamentId: 5,
  matchDate: ISODate("2024-03-20"),
  teamA: "NaVi",
  teamB: "FaZe",
  winnerTeam: "NaVi",
  records: [                        ← 嵌套的 MatchRecord 数据
    {
      matchRecordId: 201,
      map: "Mirage",
      playerStats: [                ← 嵌套的 PlayerMatchStats
        { playerId: 1, kills: 24, deaths: 14, rating: 1.42 },
        ...
      ]
    }
  ]
}
```

这种设计意味着一个 MongoDB 文档可以包含 Oracle 需要通过外键连接 3~4 张表才能得到的数据。

### 9.2 浏览如何工作（文档扫描 + 字段提取）

```java
@Override
public TableData browse(TableSpec spec, String search) {
    String table = spec.tableName();
    List<ColumnSpec> columns = spec.columns();
    String trimmed = (search == null ? "" : search.trim().toLowerCase());

    List<Object[]> rows = new ArrayList<>();
    Iterable<Document> docs = docsFor(table); // 将表名映射到 MongoDB 查询

    for (Document doc : docs) {
        Object[] row = new Object[columns.size()];
        for (int i = 0; i < columns.size(); i++) {
            row[i] = extractField(doc, columns.get(i).name(), table);
        }

        // 文本搜索：只有某个字段包含搜索词的行才保留
        if (!trimmed.isEmpty()) {
            boolean match = false;
            for (Object v : row) {
                if (v != null && v.toString().toLowerCase().contains(trimmed)) {
                    match = true;
                    break;
                }
            }
            if (!match) continue;
        }
        rows.add(row);
    }
    return new TableData(spec.columnLabels(), rows);
}
```

与 Oracle 方式（WHERE 子句在数据库端过滤）不同，MongoDB 方式**先拉取所有文档，再在 Java 中过滤**。对于小数据集（几百行）这足够快。数据量很大时效率会下降。

### 9.3 搜索如何工作

MongoDB 中的搜索是简单的 Java 端 `String.contains()` 检查：
```java
v.toString().toLowerCase().contains(trimmed)
```
这等价于 SQL 的 `LOWER(col) LIKE '%term%'`。如果文档任意一个提取字段包含搜索词，该行就被保留。

### 9.4 字段映射：Oracle 名称 → MongoDB 名称

Oracle 使用 PascalCase 列名（`PersonID`、`FullName`），MongoDB 使用 camelCase 字段名（`_id`、`fullName`）。`extractField()` 通过一个大型 switch 处理这种转换：

```java
private Object extractField(Document doc, String colName, String table) {
    return switch (table + "." + colName) {
        // People 表
        case "People.PersonID"    -> doc.get("_id");           // _id 存储 Long
        case "People.Nickname"    -> doc.get("nickname");
        case "People.FullName"    -> doc.get("fullName");
        case "People.Birthday"    -> fmt(doc.get("birthday"));  // Date → "yyyy-MM-dd"
        case "People.Nationality" -> doc.get("nationality");
        // Player 表（嵌入在 People 文档中）
        case "Player.PersonID"    -> doc.get("_id");
        case "Player.Rating"      -> nested(doc, "playerStats", "rating");
        // ...更多映射...
        default -> doc.get(colName);  // 兜底：直接用 Oracle 字段名尝试
    };
}
```

`nested(doc, parent, child)` 安全地读取两层嵌套路径：
```java
private static Object nested(Document doc, String parent, String child) {
    Object p = doc.get(parent);
    if (p instanceof Document pd) return pd.get(child);
    return null;  // 父字段不存在或不是 Document 类型
}
```

### 9.5 嵌套数组展开 — expandRecords、expandPlayerStats

某些 Oracle 表对应 MongoDB 文档内的嵌套数组。例如，Oracle 中 `MatchRecord` 是独立表；MongoDB 中它是每个 match 文档内的 `records[]` 数组。`docsFor("MatchRecord")` 调用 `expandRecords()`：

```java
private Iterable<Document> expandRecords() {
    List<Document> result = new ArrayList<>();
    for (Document match : col("matches").find()) {
        Object matchId = match.get("_id");
        List<?> records = (List<?>) match.get("records");
        if (records == null) continue;
        for (Object r : records) {
            if (r instanceof Document rec) {
                // 将数组元素展开为平铺文档，并注入父文档的 MatchID
                Document flat = new Document(rec).append("MatchID", matchId);
                result.add(flat);
            }
        }
    }
    return result;
}
```

这种**反规范化**将 MongoDB 的嵌套结构转回平铺的行列表，使通用的 `browse()` 方法能正常处理它，就好像它们是 Oracle 表中的行一样。

### 9.6 写操作 — INSERT / UPDATE / DELETE

MongoDB 写操作不使用 SQL，而是使用 MongoDB Java Driver 的流式 API。

**插入队伍：**
```java
case "Team" -> {
    String name = rawValues.get("TeamName");
    requireField(name, "Team Name", 100);
    col("teams").insertOne(
        new Document("_id", name).append("region", rawValues.get("Region"))
    );
}
```
在 teams 集合中，`_id` 字段是队伍名称（字符串类型），而非数字 ID。这在 MongoDB 中完全合法——`_id` 可以是任何类型。

**添加球员（更新已有人员文档）：**
```java
case "Player" -> {
    long id = requireLong(rawValues.get("PersonID"), "Person ID");
    Document stats = new Document("rating", parseDouble(rawValues.get("Rating")))
            .append("adr", parseDouble(rawValues.get("ADR")))
            .append("dpr", parseDouble(rawValues.get("DPR")));

    var res = col("people").updateOne(
        Filters.eq("_id", id),
        Updates.combine(
            Updates.set("type", "player"),
            Updates.set("playerStats", stats)
        )
    );
    if (res.getMatchedCount() == 0) {
        throw new IllegalArgumentException(
            "Person ID " + id + " not found. Add the person in People first.");
    }
}
```
因为 Player 数据嵌入在 People 文档中，"添加球员"实际上是一次 UPDATE：找到对应的 People 文档，添加/替换 `playerStats` 子文档并设置 `type = "player"`。

**删除球员（移除嵌套字段）：**
```java
case "Player" -> {
    long id = parseLongKey(primaryKey.get("PersonID"));
    col("people").updateOne(Filters.eq("_id", id),
        Updates.combine(
            Updates.unset("playerStats"),  // 移除 playerStats 子文档
            Updates.set("type", "other")   // 将类型还原为普通人员
        )
    );
}
```
`Updates.unset("playerStats")` 从文档中彻底移除该字段。该人员仍然存在于 `people` 集合，只是不再有球员统计数据。

### 9.7 distinctValues — Codec 问题与修复方案

原始代码调用：
```java
col(mongoCol).distinct(mongoField, Object.class)
```

MongoDB Java Driver 5.x 要求注册了对应 **Codec** 才能反序列化查询结果。`Object.class` 没有注册 Codec，运行时抛出：
```
Can't find codec for CCodecCacheKey(Class java.lang.Object, type = null)
```

修复方案是手动遍历文档，在 Java 中提取字段：

```java
@Override
public List<String> distinctValues(String table, String column) {
    String mongoCol = mongoCollectionFor(table);
    String mongoField = mongoFieldFor(table, column);

    Set<String> seen = new java.util.LinkedHashSet<>();
    for (Document doc : col(mongoCol).find()) {
        Object val = extractNestedValue(doc, mongoField);
        if (val != null) seen.add(fmtKey(val));
    }
    List<String> result = new ArrayList<>(seen);
    Collections.sort(result);
    return result;
}
```

`extractNestedValue()` 支持点路径字段，如 `"contract.teamName"`：
```java
private static Object extractNestedValue(Document doc, String fieldPath) {
    String[] parts = fieldPath.split("\\.", 2);  // 只在第一个点处分割
    Object val = doc.get(parts[0]);
    if (parts.length == 1 || !(val instanceof Document nested)) return val;
    return extractNestedValue(nested, parts[1]);  // 递归进入子文档
}
```

`mongoFieldFor()` 将 Oracle 列名映射到 MongoDB 字段名（处理 camelCase）：
```java
private static String mongoFieldFor(String table, String column) {
```

## 10. 分析层 — AnalyticsQuery、AnalyticsCatalog

### AnalyticsQuery

```java
public record AnalyticsQuery(
    String title,        // 下拉框中显示，如 "Top players by match performance"
    String description,  // 在下拉框下方以说明文字显示
    String sql,          // 实际 SQL（可能含 "?" 占位符）
    String paramLabel    // 非 null 时，显示文本输入框让用户输入筛选条件
) {
    public boolean hasParameter() { return paramLabel != null && !paramLabel.isBlank(); }

    public int placeholderCount() {
        // 统计 SQL 中 "?" 的数量——所有占位符绑定同一个值
        int count = 0;
        for (int i = 0; i < sql.length(); i++) {
            if (sql.charAt(i) == '?') count++;
        }
        return count;
    }
}
```

**为什么用 `placeholderCount()` 而不是单独的计数字段？** SQL 以文本块形式书写，统计字符避免了需要同步维护一个整数字段——如果修改 SQL 但忘记更新计数，就会产生 bug。

### AnalyticsCatalog

包含全部 8 个预定义查询的静态注册表：

| 查询 | SQL 技术 |
|---|---|
| 按比赛表现排名的球员 | `GROUP BY + AVG() + JOIN` |
| 队伍胜负记录 | `CASE WHEN` 内嵌 `SUM()`，`NULLIF` 防止除以零 |
| 队伍当前阵容 | 日期与 `SYSDATE` 比较，参数化 LIKE |
| 高于平均 Rating 的球员 | 相关子查询 `WHERE pl.Rating > (SELECT AVG...)` |
| 锦标赛参赛与名次 | 多表 `JOIN` |
| 地图统计 | `GROUP BY Map + COUNT(*) + MAX(RecordDate)` |
| 教练胜率排名 | 简单 `JOIN + ORDER BY` |
| 两队交手记录 | WHERE 含两个占位符，同一值绑定两次 |

**仅限 Oracle：** `MongoCounterStrikeRepository` 中的 `runAnalytics()` 抛出 `UnsupportedOperationException`。`AnalyticsPanel` 检测 `!repository.supportsAnalytics()`，在 MongoDB 模式下显示提示信息而不是查询表格。

### 11.2 SwingWorker — 后台线程模型

Swing 使用单一的 **EDT（事件派发线程）** 处理所有 UI 操作。如果在 EDT 上执行数据库查询，整个窗口会冻结直到查询返回——按钮停止响应，窗口可能显示为无响应状态。

**规则：所有数据库调用必须在后台线程运行；所有 UI 更新必须在 EDT 上运行。**

`SwingWorker<T, V>` 强制实施这一模式：

```java
private void loadTable(JTable table, String page, TableLoader loader) {
    String search = searchField.getText();
    setStatus("Loading " + page + "...");

    new SwingWorker<TableData, Void>() {

        @Override
        protected TableData doInBackground() throws Exception {
            // 在 SwingWorker 线程池的后台线程上运行
            return loader.load(search);  // 调用 repository——JDBC / MongoDB I/O 在这里
        }

        @Override
        protected void done() {
            // doInBackground() 完成后自动调度回 EDT
            try {
                TableData data = get();        // get() 重新抛出后台线程的异常
                setTableData(table, data);     // 安全——现在在 EDT 上
                setStatus(page + " | rows: " + data.rowCount());
            } catch (Exception exception) {
                showQueryError(exception);     // 在 EDT 上显示错误对话框
            }
        }

    }.execute();  // 将 doInBackground() 提交到工作线程池
}
```

`loader` 是一个 `@FunctionalInterface`：
```java
@FunctionalInterface
private interface TableLoader {
    TableData load(String search) throws Exception;
}
```

这使得可以将 `repository::findTopPlayers` 作为方法引用传入——实际方法在 `doInBackground()` 中的后台线程上被调用。

### 11.3 ReadOnlyTableModel — 向 JTable 提供数据

**文件：** `src/main/java/com/counterstrike/app/ui/ReadOnlyTableModel.java`

Swing 的 `JTable` 需要 `TableModel` 来提供数据。`ReadOnlyTableModel` 继承 `AbstractTableModel` 并包装一个 `TableData`：

```java
public final class ReadOnlyTableModel extends AbstractTableModel {
    private String[] columns = new String[0];
    private Object[][] rows   = new Object[0][0];

    public void setData(TableData data) {
        columns = data.columns();
        rows    = data.rows().toArray(Object[][]::new);
        fireTableStructureChanged();  // 通知 JTable 重新渲染所有内容
    }

    @Override public int     getRowCount()                    { return rows.length; }
    @Override public int     getColumnCount()                 { return columns.length; }
    @Override public String  getColumnName(int col)           { return columns[col]; }
    @Override public Object  getValueAt(int row, int col)     { return rows[row][col]; }
    @Override public boolean isCellEditable(int row, int col) { return false; } // 只读
}
```

`isCellEditable` 对所有单元格返回 `false`，防止用户直接在表格中输入——所有编辑必须通过添加/编辑表单对话框进行。

`fireTableStructureChanged()` 告知 Swing 表格列数可能发生了变化（不仅仅是数据），在切换列数不同的表格时（如从 5 列的 People 切换到 4 列的 Player）是必须的。

**`TableRowSorter`（排序 + 过滤）：**
调用 `JTable.setAutoCreateRowSorter(true)` 后，Swing 自动创建 `TableRowSorter<ReadOnlyTableModel>`。点击列头可以排序行，不需要修改底层模型。`CrudPanel` 的列过滤器在同一个 sorter 上安装 `RowFilter`。

#### 列过滤器 — updateFilterOperators()

```java
private void updateFilterOperators() {
    int idx = filterCol.getSelectedIndex();
    ColumnSpec.Type type = (idx >= 0 && idx < currentColumns.size())
            ? currentColumns.get(idx).type() : ColumnSpec.Type.TEXT;

    String[] ops = switch (type) {
        case TEXT    -> new String[]{"contains", "=", "≠"};
        case DATE    -> new String[]{"=", "≠", ">=", "<=", ">", "<"};
        default      -> new String[]{">", "<", ">=", "<=", "=", "≠"};  // INTEGER, DECIMAL
    };

    filterOp.removeAllItems();
    for (String op : ops) filterOp.addItem(op);
}
```

用户切换筛选列时，此方法替换运算符下拉框的选项。文本列只显示 `contains / = / ≠`，因为对任意字符串使用 `>` / `<` 会造成混淆。日期列和数字列显示全部比较运算符。

#### 列过滤器 — applyFilter()

```java
private void applyFilter() {
    int colIdx = filterCol.getSelectedIndex();
    String op  = (String) filterOp.getSelectedItem();
    String val = filterVal.getText().trim();

    // ── 输入校验 ─────────────────────────────────────────────────────────
    ColumnSpec.Type colType = currentColumns.get(colIdx).type();

    if (colType == INTEGER || colType == DECIMAL) {
        try { Double.parseDouble(val); }
        catch (NumberFormatException e) {
            statusLabel.setText("Invalid filter: \"" + val + "\" is not a number.");
            return;  // 不执行过滤
        }
    }
    if (colType == DATE) {
        if (!val.matches("\\d{4}-\\d{2}-\\d{2}")) {
            statusLabel.setText("Invalid filter: date must be YYYY-MM-DD.");
            return;
        }
    }

    // ── 安装 RowFilter ────────────────────────────────────────────────────
    var sorter = (TableRowSorter<ReadOnlyTableModel>) table.getRowSorter();
    sorter.setRowFilter(new RowFilter<>() {
        @Override
        public boolean include(Entry entry) {
            Object cell = entry.getValue(colIdx);
            if (cell == null) return false;
            String cellStr = cell.toString();

            if (colType == DATE) {
                // Oracle Timestamp："1990-05-15 00:00:00.0" → 截取为 "1990-05-15"
                String cellDate = cellStr.length() > 10 && cellStr.charAt(10) == ' '
                        ? cellStr.substring(0, 10) : cellStr;
                int cmp = cellDate.compareTo(val);
                return switch (op) {
                    case ">"  -> cmp >  0;
                    case "<"  -> cmp <  0;
                    case ">=" -> cmp >= 0;
                    case "<=" -> cmp <= 0;
                    case "="  -> cmp == 0;
                    case "≠"  -> cmp != 0;
                    default   -> cellDate.contains(val);
                };
            }

            if (colType == INTEGER || colType == DECIMAL) {
                double cellNum   = Double.parseDouble(cellStr);
                double filterNum = Double.parseDouble(val);
                return switch (op) {
                    case ">"  -> cellNum >  filterNum;
                    case "<"  -> cellNum <  filterNum;
                    case ">=" -> cellNum >= filterNum;
                    case "<=" -> cellNum <= filterNum;
                    case "="  -> cellNum == filterNum;
                    case "≠"  -> cellNum != filterNum;
                    default   -> cellStr.toLowerCase().contains(val.toLowerCase());
                };
            }

            // TEXT
            return switch (op) {
                case "="  -> cellStr.equalsIgnoreCase(val);
                case "≠"  -> !cellStr.equalsIgnoreCase(val);
                default   -> cellStr.toLowerCase().contains(val.toLowerCase());
            };
        }
    });
}
```

**关键点——`RowFilter` 是客户端过滤：** 过滤器不发起新的数据库查询。所有行已经在 `ReadOnlyTableModel` 的内存中。Swing 的 `TableRowSorter` 对每一行调用 `RowFilter.include()`，返回 `false` 的行被隐藏（但不从模型中删除）。调用 `clearFilter()` 即执行 `sorter.setRowFilter(null)`，所有行立刻重新可见。

**为什么日期比较用 `String.compareTo()` 而不解析为 `LocalDate`：** 以 "yyyy-MM-dd" 格式存储的日期字符串在字典序上是可排序的——`"2024-01-15".compareTo("2024-01-01")` 的结果正确为正数。将其解析为 `LocalDate` 对象也可以，但字符串比较更简单且同样正确。

#### openForm() — 动态生成添加/编辑表单

```java
private void openForm(boolean edit) {
    // 第一步：后台线程——为所有外键列获取下拉选项
    new SwingWorker<Map<String, List<String>>, Void>() {
        @Override
        protected Map<String, List<String>> doInBackground() throws Exception {
            Map<String, List<String>> options = new LinkedHashMap<>();
            for (ColumnSpec column : spec.columns()) {
                if (column.foreignKey() != null) {
                    options.put(column.name(),
                        repository.distinctValues(
                            column.foreignKey().table(),
                            column.foreignKey().column()));
                }
            }
            return options;
        }
        @Override
        protected void done() {
            showForm(spec, edit, existing, primaryKey, get());
        }
    }.execute();
}
```

```java
private void showForm(TableSpec spec, boolean edit, ...) {
    for (ColumnSpec column : spec.columns()) {
        // 添加标签："Rating" 或 "Birthday (YYYY-MM-DD)" 或 "Player ID *"
        form.add(new JLabel(column.label()
                + (column.required() ? " *" : "")
                + (column.isDate() ? " (YYYY-MM-DD)" : "")), gc);

        JComponent field;
        boolean locked = edit && column.primaryKey();  // 编辑时主键不可修改

        if (column.foreignKey() != null && !locked) {
            // 外键列 → 带已有值的可编辑下拉框
            JComboBox<String> combo = new JComboBox<>();
            combo.setEditable(true);
            for (String opt : fkOptions.getOrDefault(column.name(), List.of())) {
                combo.addItem(opt);
            }
            field = combo;
        } else {
            // 普通列 → 文本输入框
            JTextField text = new JTextField(initial, 22);
            text.setEnabled(!locked);  // 编辑模式下主键显示为灰色不可编辑
            field = text;
        }
        fields.put(column.name(), field);
    }
    // 显示对话框，收集值，调用 save()
}
```

```
  → SQL：UPDATE Player SET Rating = ?, ADR = ?, DPR = ? WHERE PersonID = ?

  → 绑定 WHERE：PersonID=7
  → executeWrite() → commit()
  → affected=1 → 成功
```

### 12.5 列过滤器（客户端）

```
Manage 页面显示 People 表（200 行已加载到内存）
用户选择 "Birthday" 列，运算符 "="，输入 "1990-05-15"，点击 Apply
  ↓
applyFilter()
  colType = DATE
  校验："1990-05-15".matches("\\d{4}-\\d{2}-\\d{2}") → 通过
  ↓
在 TableRowSorter 上安装 RowFilter：
  对模型中每一行执行：
    cell = row[3]（Birthday 列，索引 3）
    cellStr = "1990-05-15 00:00:00.0"（Oracle Timestamp.toString()）
    cellStr.charAt(10) == ' ' → true
    cellDate = "1990-05-15"（前 10 位）
    op = "=" → cellDate.compareTo("1990-05-15") == 0 → true → 保留此行

    其他行：cellDate ≠ "1990-05-15" → 排除
  ↓
JTable 隐藏所有不匹配的行
不发起数据库查询——所有数据已在内存中
状态栏："Filter: Birthday = 1990-05-15  →  1 rows"
```

## 13. 设计原则总结

| 原则 | 应用位置 | 好处 |
|---|---|---|
| **接口隔离** | `AppRepository` 接口 | UI 不导入任何 Oracle 或 MongoDB 类 |
| **元数据驱动** | `Schema`、`TableSpec`、`ColumnSpec` | 添加新表无需写 SQL 也无需改 UI 代码 |
| **PreparedStatement** | 所有 JDBC 查询 | 用户输入无法触发 SQL 注入 |
| **显式事务** | `executeWrite()` | INSERT/UPDATE/DELETE 出错自动回滚 |
| **后台线程** | 所有 `SwingWorker` 使用处 | 数据库 I/O 期间 UI 永不冻结 |
| **快速失败** | 启动时 `testConnection()` | 错误在 UI 构建前就暴露 |
| **不可变 Record** | `TableData`、`TableSpec`、`HomeSummary` | 共享数据不会被意外修改 |
| **客户端过滤** | `CrudPanel` 中的 `RowFilter` | 列过滤器即时生效，无额外数据库请求 |
| **懒加载 DB 连接** | `MongoDatabase.getDatabase()` | MongoDB 连接池在首次使用前不创建 |
| **类型安全绑定** | Repository 中的 `bind()` | 日期和数字在 SQL 中类型正确 |
