# CS2 数据库浏览器 — 源代码说明文档

本文档按功能模块逐层解释关键代码的作用，包括**为什么能搜索、为什么能查询、为什么能添加/编辑/删除**。

## 一、整体架构

```
程序入口
  CounterStrikeApp.java          启动、选数据库、建连接

数据库连接层（db/）
  Database.java                  Oracle JDBC 连接管理
  MongoDatabase.java             MongoDB 连接管理

数据访问层（repository/）
  AppRepository.java             接口：定义所有数据操作的"合同"
  CounterStrikeRepository.java   Oracle 实现
  MongoCounterStrikeRepository.java  MongoDB 实现
  Schema.java                    所有表的字段定义（元数据）
  TableSpec / ColumnSpec         表结构 / 列结构的描述对象

UI 层（ui/）
  MainFrame.java                 主窗口框架、导航栏
  CrudPanel.java                 Manage 页：增删改查 + 列过滤
  AnalyticsPanel.java            Analytics 页：高级 SQL 查询
  Styles.java                    按钮、表格的通用样式工具
```

程序的核心设计思路是**接口隔离**：UI 层只认识 `AppRepository` 这个接口，不管底下是 Oracle 还是 MongoDB。切换数据库时，只要换一个实现类传进去，界面完全不用改。

## 二、程序入口：CounterStrikeApp.java

```java
public static void connect(javax.swing.JFrame oldFrame) {
    // 弹出对话框让用户选 Oracle 或 MongoDB
    int choice = JOptionPane.showOptionDialog(...);

    if (choice == 0) {
        // Oracle 路径
        DatabaseConfig config = DatabaseConfig.load();   // 读 db.properties 配置文件
        Database database = new Database(config);
        database.testConnection();                       // 试连接，失败直接报错
        repository = new CounterStrikeRepository(database);
    } else {
        // MongoDB 路径
        MongoDatabase mongo = new MongoDatabase(uri, "cs_small");
        mongo.testConnection();                         // 发一个 ping 命令验证连接
        repository = new MongoCounterStrikeRepository(mongo);
    }

    MainFrame frame = new MainFrame(repository, dbInfo);
    frame.setVisible(true);
    if (oldFrame != null) oldFrame.dispose();  // 关掉旧窗口（Switch DB 用）
}
```

**关键点**：`connect()` 是 `public static`，所以主窗口里的"Switch DB"按钮可以随时调用它，传入当前窗口作为 `oldFrame`，连接成功后新窗口开、旧窗口关，实现热切换。

## 三、数据库连接层

### Database.java（Oracle）

```java
public Connection getConnection() throws SQLException {
    loadOracleDriverIfPresent();
    return DriverManager.getConnection(config.url(), config.user(), config.password());
}
```

- **每次调用都开一个新连接**。Oracle JDBC 的 `DriverManager.getConnection()` 是标准 Java 接口，传入 URL（`jdbc:oracle:thin:@localhost:1521/FREE`）、用户名、密码即可。
- `loadOracleDriverIfPresent()` 手动加载 `oracle.jdbc.OracleDriver`，保证在旧 JVM 上也能找到驱动。
- `testConnection()` 只是开一个连接再立刻关掉，用来验证配置是否正确。

### MongoDatabase.java（MongoDB）

```java
public com.mongodb.client.MongoDatabase getDatabase() {
    if (client == null) {
        client = MongoClients.create(uri);  // 第一次调用才真正建连接池
    }
    return client.getDatabase(dbName);     // 返回 cs_small 数据库对象
}

public void testConnection() {
    getDatabase().runCommand(new Document("ping", 1));  // 向服务器发 ping
}
```

- `MongoClients.create(uri)` 创建连接池，之后所有请求复用这一个池（懒加载）。
- `getDatabase("cs_small")` 不真正发请求，只是拿到数据库句柄。
- `close()` 关闭连接池，用于 Switch DB 时防止连接泄漏。

## 四、元数据层：Schema / TableSpec / ColumnSpec

这是整个系统的**核心设计**。所有表的结构用 Java 对象描述，UI 和 SQL 都从这里自动生成，不需要为每张表单独写界面或 SQL。

### ColumnSpec.java — 描述一列

```java
public record ColumnSpec(
    String name,        // 数据库字段名，如 "PersonID"
    String label,       // 界面显示名，如 "Person ID"
    Type type,          // TEXT / INTEGER / DECIMAL / DATE
    boolean primaryKey, // 是否主键（主键在编辑时不可改）
    boolean required,   // 是否必填
    ForeignKey foreignKey  // 外键引用（用于生成下拉框）
)
```

工厂方法简化写法：
- `pk(...)` — 主键列
- `col(...)` — 普通可选列
- `required(...)` — 必填列
- `ref(...)` — 有外键引用的列（界面显示下拉框）

### TableSpec.java — 描述一张表

```java
public record TableSpec(
    String displayName,   // 界面显示名，如 "Player Contract"
    String tableName,     // 真实表名，如 "PlayerContract"
    List<ColumnSpec> columns,
    String orderBy        // 默认排序，如 "MatchDate DESC NULLS LAST"
)
```

三个关键方法：
- `primaryKey()` — 返回主键列列表，用于 WHERE 子句定位行
- `insertableColumns()` — 插入时用到的所有列
- `editableColumns()` — 编辑时只改非主键列

### Schema.java — 全部表的目录

```java
public static final TableSpec PEOPLE = new TableSpec("People", "People", List.of(
    pk("PersonID", "Person ID", Type.INTEGER),
    required("Nickname", "Nickname", Type.TEXT),
    required("FullName", "Full Name", Type.TEXT),
    col("Birthday", "Birthday", Type.DATE),
    ref("Nationality", "Nationality", Type.TEXT, "People", "Nationality", false)
), "PersonID");
```

`ref("Nationality", ..., "People", "Nationality", false)` 表示：Nationality 的下拉选项来自 People 表自身的 Nationality 列（自引用），这样添加球员时可以从已有国籍中选择，也可以手动输入新国籍。

## 六、Oracle 实现：CounterStrikeRepository.java

### 6.1 为什么能搜索（顶部搜索栏）

以球员搜索为例：

```java
public TableData findTopPlayers(String search) throws SQLException {
    String sql = """
        SELECT p.PersonID, p.Nickname, p.FullName, pl.Rating, pl.ADR, pl.DPR
        FROM People p
        JOIN Player pl ON p.PersonID = pl.PersonID
        WHERE LOWER(p.FullName) LIKE ?
           OR LOWER(NVL(p.Nickname, '')) LIKE ?
           OR TO_CHAR(pl.Rating) LIKE ?
        ORDER BY pl.Rating DESC NULLS LAST
        """;
    String textLike = like(search);   // 把 "s1mple" 变成 "%s1mple%"
    return queryTable(columns, sql, statement -> {
        statement.setString(1, textLike);
        ...
    });
}
```

- `like(search)` 把用户输入变成 `%输入内容%`，Oracle 的 `LIKE` 运算符就能模糊匹配。
- `LOWER()` 让搜索不区分大小写。
- `NVL(Nickname, '')` 处理 NULL 值，避免 NULL 直接比较导致不匹配。
- `?` 是占位符，防止 SQL 注入攻击（用户输的内容不会当成 SQL 执行）。

### 6.2 为什么能查询（Manage 页面的浏览）

`browse()` 方法根据 `TableSpec` 元数据**自动生成 SQL**，不用为每张表单独写：

```java
public TableData browse(TableSpec spec, String search) throws SQLException {
    // 1. 自动拼 SELECT 列名
    StringJoiner select = new StringJoiner(", ");
    for (ColumnSpec column : spec.columns()) {
        select.add(column.name());
    }

    // 2. 拼 WHERE 子句（日期列用 TO_CHAR 转成字符串再 LIKE）
    StringJoiner where = new StringJoiner(" OR ", " WHERE ", "");
    for (ColumnSpec column : columns) {
        if (column.isDate()) {
            where.add("TO_CHAR(" + column.name() + ", 'YYYY-MM-DD') LIKE ?");
        } else {
            where.add("LOWER(TO_CHAR(" + column.name() + ")) LIKE ?");
        }
    }

    // 3. 拼 ORDER BY
    sql.append(" ORDER BY ").append(spec.orderBy());

    // 最终 SQL 类似：
    // SELECT PersonID, Nickname, FullName, Birthday, Nationality
    // FROM People
    // WHERE LOWER(TO_CHAR(PersonID)) LIKE ?
    //    OR LOWER(TO_CHAR(Nickname)) LIKE ?
    //    OR ...
    // ORDER BY PersonID
}
```

**任意新增一张表，只要在 Schema.java 里加一个 `TableSpec`，browse() 自动支持，不用多写一行 SQL。**

### 6.3 为什么能添加（INSERT）

```java
public void insertRow(TableSpec spec, Map<String, String> rawValues) throws SQLException {
    List<ColumnSpec> columns = spec.insertableColumns();

    // 自动拼：INSERT INTO People (PersonID, Nickname, ...) VALUES (?, ?, ...)
    StringJoiner names = new StringJoiner(", ");
    StringJoiner marks = new StringJoiner(", ");
    for (ColumnSpec column : columns) {
        names.add(column.name());
        marks.add("?");
    }
    String sql = "INSERT INTO " + spec.tableName() + " (" + names + ") VALUES (" + marks + ")";

    executeWrite(sql, statement -> {
        int index = 1;
        for (ColumnSpec column : columns) {
            bind(statement, index++, column, rawValues.get(column.name()));
        }
    });
}
```

`bind()` 根据列类型把字符串转为正确的 Java 类型再绑定：
- `Type.INTEGER` → `statement.setLong()`
- `Type.DECIMAL` → `statement.setBigDecimal()`
- `Type.DATE` → `statement.setDate()`（先把 "2003-05-15" 解析成 `LocalDate`）
- `Type.TEXT` → `statement.setString()`

`executeWrite()` 里开启了**显式事务**（`setAutoCommit(false)`），插入成功才 `commit()`，失败自动 `rollback()`，保证数据一致性。

### 6.4 为什么能编辑（UPDATE）

```java
public void updateRow(TableSpec spec, Map<String, String> primaryKey,
                      Map<String, String> rawValues) throws SQLException {
    List<ColumnSpec> editable = spec.editableColumns();  // 主键不在这里

    // 拼：UPDATE People SET Nickname = ?, FullName = ?, ... WHERE PersonID = ?
    StringJoiner set = new StringJoiner(", ");
    for (ColumnSpec col : editable) set.add(col.name() + " = ?");

    StringJoiner where = new StringJoiner(" AND ");
    for (ColumnSpec key : spec.primaryKey()) where.add(key.name() + " = ?");

    String sql = "UPDATE " + spec.tableName() + " SET " + set + " WHERE " + where;
    int affected = executeWrite(sql, ...);

    if (affected == 0) {
        throw new IllegalArgumentException("No matching row was found to update.");
    }
}
```

`primaryKey` 是一个 `Map`，存的是主键列名和原值（从表格里读出来的），用于 WHERE 条件精确定位那一行。

### 6.5 为什么能删除（DELETE）

```java
public void deleteRow(TableSpec spec, Map<String, String> primaryKey) throws SQLException {
    StringJoiner where = new StringJoiner(" AND ");
    for (ColumnSpec key : spec.primaryKey()) {
        where.add(key.name() + " = ?");
    }
    String sql = "DELETE FROM " + spec.tableName() + " WHERE " + where;
    int affected = executeWrite(sql, ...);

    if (affected == 0) {
        throw new IllegalArgumentException("No matching row was found to delete.");
    }
}
```

逻辑和 UPDATE 类似，用主键定位行，`affected == 0` 说明行不存在（已被别人删了），给用户提示。

```java
               + " WHERE " + column + " IS NOT NULL ORDER BY " + column;
```

## 七、MongoDB 实现：MongoCounterStrikeRepository.java

MongoDB 里没有表，数据以**集合（Collection）+ 嵌套文档**存储。`cs_small` 数据库的结构：

| Oracle 表 | MongoDB 集合 | 存储方式 |
|---|---|---|
| People, Player, Coach, Contract | `people` | 球员基本信息 + 嵌套的 playerStats、contract |
| Team | `teams` | 简单文档，`_id` 是队名 |
| Tournament | `tournaments` | 嵌套 `participation[]` 数组 |
| MatchTable, MatchRecord | `matches` | 嵌套 `records[]` + `playerStats[]` |

### 7.1 为什么能查询

```java
@Override
public TableData browse(TableSpec spec, String search) {
    Iterable<Document> docs = docsFor(spec.tableName());  // 拿到对应文档集合
    for (Document doc : docs) {
        Object[] row = new Object[columns.size()];
        for (int i = 0; i < columns.size(); i++) {
            row[i] = extractField(doc, columns.get(i).name(), spec.tableName());
        }
        // 搜索过滤：检查任意字段是否包含搜索词
        if (!trimmed.isEmpty()) {
            boolean match = false;
            for (Object v : row) {
                if (v != null && v.toString().toLowerCase().contains(trimmed)) {
                    match = true; break;
                }
            }
            if (!match) continue;
        }
        rows.add(row);
    }
}
```

`docsFor()` 把 Oracle 表名映射到 MongoDB 查询：
```java
private Iterable<Document> docsFor(String table) {
    return switch (table) {
        case "People" -> col("people").find();
        case "Player" -> col("people").find(Filters.eq("type", "player"));
        case "Coach"  -> col("people").find(Filters.eq("type", "coach"));
        case "Team"   -> col("teams").find();
        // ...
        case "MatchRecord" -> expandRecords();  // 展开嵌套数组
    };
}
```

`extractField()` 把 Oracle 的列名映射到 MongoDB 的字段路径，例如：
```java
case "Player.Rating" -> nested(doc, "playerStats", "rating");
// 等价于读 doc["playerStats"]["rating"]
```

### 7.2 为什么能添加

以添加球员（Player）为例：

```java
case "Player" -> {
    long id = requireLong(rawValues.get("PersonID"), "Person ID");
    Document stats = new Document("rating", parseDouble(rawValues.get("Rating")))
            .append("adr", parseDouble(rawValues.get("ADR")))
            .append("dpr", parseDouble(rawValues.get("DPR")));

        Updates.combine(

        throw new IllegalArgumentException("Person ID " + id + " not found.");
    }
}
```

MongoDB 里 Player 数据**内嵌**在 People 文档里，所以"添加球员"实际上是在已有人员文档上追加字段。

### 7.3 distinctValues 的 MongoDB 实现

```java
@Override
public List<String> distinctValues(String table, String column) {
    String mongoCol = mongoCollectionFor(table);   // 映射集合名
    String mongoField = mongoFieldFor(table, column);  // 映射字段名（处理 camelCase）

    Set<String> seen = new LinkedHashSet<>();
    for (Document doc : col(mongoCol).find()) {
        Object val = extractNestedValue(doc, mongoField);  // 支持 "contract.teamName" 点路径
        if (val != null) seen.add(fmtKey(val));
    }
    List<String> result = new ArrayList<>(seen);
    Collections.sort(result);
    return result;
}
```

注意这里用的是 `find()` 全量扫描而不是 `distinct()`，原因是 MongoDB Java Driver 的 `distinct(field, Object.class)` 有 Codec 注册问题会抛出运行时错误，手动遍历可以绕过这个限制。

**列过滤器的工作原理**：

```java
private void updateFilterOperators() {
    // 根据当前列的类型决定显示什么运算符
    ColumnSpec.Type type = currentColumns.get(filterCol.getSelectedIndex()).type();
    String[] ops = switch (type) {
        case TEXT    -> new String[]{"contains", "=", "≠"};
        case DATE    -> new String[]{"=", "≠", ">=", "<=", ">", "<"};
        default      -> new String[]{">", "<", ">=", "<=", "=", "≠"};
    };

```

`RowFilter` 决定表格哪些行显示：

```java
sorter.setRowFilter(new RowFilter<>() {
    @Override
    public boolean include(Entry entry) {
        Object cell = entry.getValue(colIdx);
        String cellStr = cell.toString();

        if (colType == DATE) {
            // 截掉 Oracle 可能带的时间部分 "1990-05-15 00:00:00.0" → "1990-05-15"
            String cellDate = cellStr.length() > 10 ? cellStr.substring(0, 10) : cellStr;
            return cellDate.compareTo(val) == 0;  // 日期字符串字典序可以直接比大小
        }

        if (colType == INTEGER || colType == DECIMAL) {
            double cellNum = Double.parseDouble(cellStr);
            double filterNum = Double.parseDouble(val);
            return cellNum > filterNum;  // 数值比较
        }

        return cellStr.toLowerCase().contains(val.toLowerCase());  // 文本模糊匹配
    }
});
```

**`RowFilter` 是纯客户端过滤**：数据已经从数据库加载到内存里，过滤只是控制哪些行显示在界面上，不会再发数据库请求。

**添加/编辑表单的生成**：

```java
private void openForm(boolean edit) {
    // 1. 后台线程：查所有外键列的下拉选项
    Map<String, List<String>> options = new LinkedHashMap<>();
    for (ColumnSpec column : spec.columns()) {
        if (column.foreignKey() != null) {
            options.put(column.name(),
                repository.distinctValues(column.foreignKey().table(),
                                          column.foreignKey().column()));
        }
    }
    // 2. 回到 EDT：根据 ColumnSpec 动态生成表单字段
    showForm(spec, edit, existing, primaryKey, options);
}

private void showForm(...) {
    for (ColumnSpec column : spec.columns()) {
        if (column.foreignKey() != null) {
            // 有外键 → 可编辑下拉框（从 distinctValues 拿选项）
            JComboBox<String> combo = new JComboBox<>();
            combo.setEditable(true);
            for (String opt : options.get(column.name())) combo.addItem(opt);
        } else {
            // 无外键 → 普通文本输入框
            JTextField text = new JTextField(initial, 22);
            text.setEnabled(!locked);  // 主键在编辑时不可修改
        }
    }
}
```

## 九、关键数据流总结

### 用户搜索一个球员

```
用户在搜索框输入 "s1mple" → 按回车
→ MainFrame.reloadCurrentPage() 触发
→ SwingWorker 后台线程执行
    → CounterStrikeRepository.findTopPlayers("s1mple")
    → 生成 SQL: ... WHERE LOWER(p.FullName) LIKE '%s1mple%' OR ...
    → JDBC 发送到 Oracle，拿回 ResultSet
    → 打包成 TableData(列名[], 行数据[])
→ 回到 EDT
→ ReadOnlyTableModel.setData(tableData) 更新 JTable
→ 界面刷新，显示匹配的行
```

### 用户在 Manage 添加一个球员

```
用户点 Add → openForm(false)
→ SwingWorker 后台线程：
    → 查 Player.PersonID 的外键 → distinctValues("People", "PersonID")
    → 拿到所有人员 ID 列表
→ 回到 EDT，弹出表单，PersonID 显示下拉框
用户填写数据，点 OK
→ save() 调用 repository.insertRow(PLAYER_SPEC, values)
→ SwingWorker 后台线程：
    → CounterStrikeRepository.insertRow()
    → 自动生成 INSERT INTO Player (PersonID, Rating, ADR, DPR) VALUES (?, ?, ?, ?)
    → bind(): "1.05" → setDouble(1.05)
    → executeWrite(): setAutoCommit(false) → executeUpdate() → commit()
→ 回到 EDT，状态栏显示 "Row added."，表格刷新
```

### 列过滤（纯客户端，不查数据库）


## 十、为什么不用重复写 SQL / MongoDB 操作

系统的核心是**元数据驱动**：

1. 在 `Schema.java` 里加一行 `TableSpec`
2. `browse()` 自动生成对应的 SELECT
3. `insertRow()` 自动生成 INSERT
4. `updateRow()` 自动生成 UPDATE  
5. `deleteRow()` 自动生成 DELETE
6. `CrudPanel` 自动生成对应的表单界面

新增一张表不需要动 SQL、不需要动界面代码，只需要描述它有哪些字段、什么类型、哪些是主键。
